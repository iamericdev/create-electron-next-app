# Electron Next.js Application

This is a template application built with Next.js and Electron, generated by `create-electron-next-app`.

## Getting Started

First, install the dependencies if you haven't already. Note: This template currently utilizes `pnpm` under the hood for some of its `package.json` scripts, so `pnpm` is highly recommended.

```bash
pnpm install
# or
npm install
# or
yarn install
```

### Development

To start the development server for both Next.js and the Electron app simultaneously:

```bash
pnpm run dev:all
# or
npm run dev:all
# or
yarn dev:all
```

Alternatively, you can run them separately. First, start the Next.js dev server:

```bash
pnpm dev
```

Then, in another terminal, launch the Electron app:

```bash
pnpm run dev:electron
```

### Building for Production

To package the application for different platforms:

**Windows:**

```bash
pnpm run dist:win
```

**macOS:**

```bash
pnpm run dist:mac
```

**Linux:**

```bash
pnpm run dist:linux
```

_(These commands will compile the Next.js app, transpile the Electron main process, and package it using `electron-builder`)._

## Note on Package Managers

By default, the npm scripts in this template are optimized for `pnpm` (e.g., using `pnpm dev` inside `concurrently`).

If you prefer to use **npm** or **yarn**, you may want to update the scripts in your `package.json` to use your preferred package manager (e.g., changing `"dev:all": "concurrently \"pnpm dev\" \"pnpm dev:electron\""` to use `npm run dev`).

According to the [Electron documentation](https://www.electronjs.org/docs/latest/tutorial/tutorial-first-app), if you use standard `npm start` natively, Electron executes the `start` script defined in your `package.json`. Make sure your scripts accurately reflect the package manager you are using to avoid command-not-found errors during nested script execution!
